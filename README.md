# ⚡ Анонимная P2P Торговая Платформа в Telegram (Aiogram 3)

Высокопроизводительный, безопасный и масштабируемый Telegram-бот для проведения P2P-сделок обмена фиата и криптовалюты (GRAM). Архитектура построена по модульному принципу на базе **Aiogram 3.x** и **SQLite (aiosqlite)** с поддержкой асинхронного режима WAL.

---

## 🛠 Ключевая Архитектура и Модули

* **`main.py`** — Центральная точка входа. Управляет инициализацией сессий, жизненным циклом фоновых задач и координирует иерархию роутеров диспетчера (Фильтры приватных чатов, защита от утечек интерфейса).
* **`database.py`** — Ядро базы данных. Оптимизировано с помощью `PRAGMA synchronous=NORMAL` и `cache_size`. Содержит атомарные методы проверок реквизитов, лимитов ордеров и защиты от флуда.
* **`verification.py`** — Двухэтапная верификация пользователей (KYC).
* **`cabinet.py`** — Личный Кабинет пользователя. Настройка реквизитов (Банковские карты, Piastrix, TON), вывод персональной статистики и архива последних 5 сделок.
* **`offers.py`** — Торговый стакан P2P-объявлений. Сортировка предложений, динамические лимиты на создание ордеров на основе рангов.
* **`admin.py`** — Интерактивная графическая панель администратора (GUI) с пагинацией (постраничным выводом списков), автоматическим сбросом FSM-состояний и прямым переходом в пульт модерации пользователя в один клик.

## 🛡️ Безопасность и Защита Платформы

1. **100% Анонимность чата сделок:** Бот выступает виртуальным ретранслятором (`bot.send_message`). Использование встроенной функции `message.forward()` полностью исключено — ссылки на реальные аккаунты участников и метаданные медиафайлов физически не передаются контрагенту.
2. **Защита от Ботоферм (Sybil-атак):** Подача заявки на верификацию заблокирована до тех пор, пока бот не подтвердит через Telegram API (`get_chat_member`) активную подписку пользователя на закрытый канал/чат платформы.
3. **Изоляция Команд в ЛС:** Все системные команды (`/start`, `/admin`, `/debug`) жестко отфильтрованы по параметру `F.chat.type == "private"`. Бот слеп внутри групп, что исключает случайный вывод админки на публику.
4. **Конфликт интересов (Гарант сам себе):** Учетным записям администраторов на уровне кода и UI заблокирован доступ к созданию объявлений и принятию сделок в стакане.
5. **Фиксация Реквизитов:** Банковские данные продавца извлекаются из БД и фиксируются в чате покупателя в момент нажатия кнопки принятия сделки. Изменение карт в ЛК во время тиканья таймера оплаты не влияет на уже открытую сделку.
6. **Защита от Перехвата Сделок:** Внедрена атомарная проверка `guarantor_id IS NOT NULL` с Race Condition защитой. Два гаранта из комьюнити физически не могут захватить один и тот же ордер.

## 🎮 Геймификация и Репутационная Система

* **Ядро вычисления репутации:** Система считает точное среднее арифметическое на основе суммы всех полученных звезд (`rating_sum`) и общего количества голосов (`rating_count`), защищая исторический опыт трейдера от случайных просадок.
* **Схлопывание клавиатур:** Сразу после выставления оценки 1–5 звезд клавиатура отзыва уничтожается, предотвращая накрутку репутации повторными кликами за одну сделку.
* **Система 10 Динамических Рангов:** Трейдерам присваиваются звания, отображаемые в ЛК и стакане:
  `🥚 Новичок` ➔ `🐾 Местный` ➔ `🌱 Бывалый` ➔ `⚔️ Старший Трейдер` ➔ `🛡️ Проверенный` ➔ `🔥 Про-Трейдер` ➔ `⚜️ Эксперт Торговли` ➔ `⚡ Акула Трейдинга` ➔ `🔱 Легенда Платформы` ➔ `👑 Небожитель P2P`.
* **Автоматические веса рангов:** Сетки званий учитывают рабочие просадки рейтинга (оценки 4.0 - 4.3 за скорость ответа) на средних этапах объема торгов, делая прогресс честным и достижимым.

## 👥 Ролевая Модель Системы

1. **Пользователи (Трейдер):** Требует верификации, настраивает ЛК, выставляет ордера, торгует напрямую или запрашивает Гаранта.
2. **Гаранты (`guarantor_member`):** Доверенный член сообщества. Не имеет доступа к панели `/admin`, но получает алерты о сделках, депонирует средства на свои кошельки, удерживает **10% комиссии** платформы и управляет ручным выпуском/возвратом средств.
3. **Администраторы (`ADMIN_IDS`):** Полный доступ к GUI-панели управления, ручным банам (вечным/временным), выгрузке логов, просмотру KYC-документов пользователей и арбитражу споров.

## 🛠️ Технологический стек
* **Language:** Python 3.11+
* **Framework:** Aiogram 3.x (Asynchronous Telegram Bots API)
* **Database:** SQLite3 + aiosqlite (Async driver with WAL mode)
* **State Machine:** Aiogram FSM Context (управление шагами сделок и ввода реквизитов)

## ⚙️ Установка и локальный запуск

1. Клонируйте репозиторий:
```bash
git clone https://github.com/iermiloff/p2p_trade_bot/
cd p2p_trade_bot
```

2. Создайте виртуальное окружение и установите зависимости:
```bash
python3 -m venv venv
source venv/bin/activate  # Для Windows: venv\Scripts\activate
pip install -r requirements.txt
```

3. Задайте обязательные переменные окружения в .env файле:
```bash
touch .env
chmod 777 .env
nano .env
```
```env
# Токен вашего бота, полученный от @BotFather
BOT_TOKEN=8869098441:AA...ваш_токен...

# Telegram ID главных администраторов платформы (через запятую, без пробелов)
ADMIN_IDS=6912785652,123456789

# Параметры защиты от ботоферм и автоматических скриптов
REQUIRED_CHANNEL_ID=-1002447953259
CHANNEL_INVITE_LINK=https://t.me
```

> ⚠️ **Важно:** 
> 1. Числовой идентификатор закрытого канала или супергруппы `REQUIRED_CHANNEL_ID` обязательно должен начинаться с префикса `-100`.
> 2. Для корректной проверки подписки пользователей бот должен быть добавлен в этот закрытый канал/чат в качестве **Администратора** с минимальным правом на *Просмотр списка участников (View Members)*.
4. Запустите бота:
```bash
export $(xargs < .env) && python3 main.py
```

## 📦 Деплой на Linux-сервер (через systemd)
Для обеспечения работы бота 24/7 создайте файл службы `/etc/systemd/system/p2pbot.service`:
```ini
[Unit]
Description=Telegram P2P Exchange Bot
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/root/p2p_trade_bot
EnvironmentFile=/root/p2p_trade_bot/.env
ExecStart=/root/p2p_trade_bot/venv/bin/python main.py
Restart=always

[Install]
WantedBy=multi-user.target
```
5. Активируйте и запустите службу:
```bash
systemctl daemon-reload
systemctl enable p2pbot
systemctl start p2pbot
```
